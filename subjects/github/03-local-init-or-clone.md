# 本地：初始化仓库 或 克隆远程仓库

> 本地要有一个「带 .git 的仓库」才能做 commit 和 push。两种方式：自己 `git init` 新建，或用 `git clone` 从 GitHub 拷一份下来；各自在什么场景用、背后结构是什么。

- **标签**：`#Git` `#init` `#clone` `#本地仓库`
- **日期**：2025-03-11
- **来源**：Ad-Studium 学习笔记

---

## 1. 核心概念 / 定义

| 术语 | 含义 |
|------|------|
| **git init** | 在当前文件夹里创建一个新的 **.git** 目录，让这个文件夹变成「Git 仓库」。之后所有版本历史都会存在这里。 |
| **git clone &lt;URL&gt;** | 从远程（如 GitHub）**完整复制**一个已有仓库到当前目录：包括所有文件 + 整个提交历史，并自动把该远程记作 `origin`。 |

---

## 2. 要点 / 步骤

- **场景 A：本地已有项目文件夹，还没用过 Git**  
  → 进入该文件夹，执行 `git init`，再按 05 关联远程并 push。

- **场景 B：远程已经建好（或别人已有仓库），你本地还没有**  
  → 在要放项目的父目录执行 `git clone <仓库URL>`，会生成一个和仓库同名的子文件夹，里面已经是完整仓库。

---

## 3. 细节与推导

### 3.1 用 `git init` 新建本地仓库

**什么时候用**：你电脑上已经有一个项目文件夹（比如 `d:\Ad-Studium`），里面有不少文件，但还没有用 Git 管理，现在想把它变成仓库并推到 GitHub。

**指令与含义**：

```bash
cd d:\Ad-Studium
```

- **在干什么**：进入你要当成「项目根目录」的文件夹。之后 `git init` 只作用于当前目录。

```bash
git init
```

- **在干什么**：在当前目录下创建隐藏文件夹 `.git`，用来存：提交历史、分支信息、暂存区索引等。  
- **背后结构**：执行后，这个目录就变成了「工作区 + 本地仓库」；还没有任何 commit，也没有关联任何远程。  
- **结果**：终端会提示 `Initialized empty Git repository in .../Ad-Studium/.git/`。此时你还没有「第一次提交」，所以远程上如果已经建了空仓库，需要先 add、commit，再按 05 关联并 push。

**小结**：`git init` = 只做「把当前文件夹变成 Git 仓库」这一件事，不涉及远程、不涉及第一次提交。

---

### 3.2 用 `git clone` 把远程仓库拷到本地

**什么时候用**：GitHub 上已经有一个仓库（你刚建的或别人的），你希望本地有一份完整副本，包括历史和远程关联。

**指令与含义**：

```bash
cd d:\
```

- **在干什么**：进入你希望「放项目」的父目录。clone 会在这里**新建一个文件夹**，名字等于仓库名。

```bash
git clone https://github.com/你的用户名/仓库名.git
```

- **在干什么**：  
  1. 向 GitHub 请求这个仓库的所有数据（所有 commit、所有文件）。  
  2. 在当前目录下创建一个名为 `仓库名` 的文件夹。  
  3. 把文件和解出的历史放进这个文件夹，并执行一次「等价于 git init + 把远程历史全部拉下来」。  
  4. 自动添加一个叫 **origin** 的远程，指向你刚 clone 的 URL。  
- **背后结构**：clone = **init + 拉取整个历史 + 配置 origin** 的一键操作；所以你不需要再 `git init`，也不需要再 `git remote add origin ...`（除非要改地址）。  
- **结果**：本地多了一个「仓库名」文件夹，进去后直接可以改代码、add、commit、push。

**如果想把仓库 clone 到指定文件夹名**（而不是默认的仓库名）：

```bash
git clone https://github.com/用户名/仓库名.git 我的文件夹名
```

- **在干什么**：和上面一样，只是本地文件夹名改成 `我的文件夹名`，而不是 `仓库名`。

---

### 3.3 两种方式对比

| 方式 | 适用场景 | 本地状态 after | 要不要再关联远程 |
|------|----------|----------------|------------------|
| **git init** | 本地先有项目，后建 GitHub 仓库 | 空历史，只有工作区文件 | 要，用 `git remote add origin <URL>` |
| **git clone** | 远程先有仓库，本地从零拿一份 | 已有完整历史 + 自动 origin | 一般不需要，已关联 |

---

## 4. 总结 / 与其它知识的联系

- **git init**：在当前目录「挂上」Git，产生 `.git`，变成仓库；不拉取、不关联远程，适合「本地先、远程后」的流程。  
- **git clone**：从远程复制一整份仓库到本地，自动带历史和 `origin`，适合「远程先、本地后」或协作时拿别人的仓库。  
- 有了本地仓库之后，下一步就是：把修改**纳入版本**（add + commit），再**上传到 GitHub**（remote + push），见 04、05、06。
