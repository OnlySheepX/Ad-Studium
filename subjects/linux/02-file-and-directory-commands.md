# 文件与目录常用指令

> 在 Linux 里「在哪、有什么、进去、新建、复制、移动、删除」——ls / cd / pwd / mkdir / touch / cp / mv / rm 每条在干什么、背后结构是什么。

- **标签**：`#Linux` `#文件` `#目录` `#ls` `#cd` `#cp` `#mv` `#rm`
- **日期**：2025-03-11
- **来源**：Ad-Studium 学习笔记

---

## 1. 核心概念 / 定义

| 术语 | 含义 |
|------|------|
| **当前工作目录** | shell 当前「站在」的目录；相对路径、不带路径的命令都基于这里。 |
| **家目录** | 当前用户的主目录，通常 `/home/用户名`；用 `~` 表示。 |
| **隐藏文件/目录** | 名字以 `.` 开头的项；`ls` 默认不显示，需加 `-a`。 |

---

## 2. 要点 / 步骤

1. **看在哪**：`pwd`；**看有什么**：`ls`（可跟路径）；**换目录**：`cd 路径`。
2. **新建目录**：`mkdir`；**新建空文件**：`touch`。
3. **复制**：`cp 源 目标`；**移动/改名**：`mv 源 目标`；**删除**：`rm`（慎用，默认不进回收站）。

---

## 3. 细节与推导

### 3.1 定位与查看

**pwd（Print Working Directory）**

```bash
pwd
```

- **在干什么**：输出**当前工作目录**的绝对路径。  
- **背后结构**：shell 内部维护一个「当前目录」；所有相对路径都从这个目录开始解析。

**ls（List）**

```bash
ls
ls -l
ls -a
ls -la   # 或 ls -l -a
ls /etc
```

- **在干什么**：  
  - `ls`：列出**当前目录**下的文件名（不含隐藏）。  
  - `ls -l`：以「长格式」列出，包含权限、所有者、大小、修改时间等。  
  - `ls -a`：**全部**列出，包含以 `.` 开头的隐藏文件/目录。  
  - `ls /etc`：列出**指定路径**下的内容，不改变当前目录。  
- **背后结构**：ls 读的是目录文件里的「目录项」；权限、大小等来自文件的 inode 与元数据。

**cd（Change Directory）**

```bash
cd /home/user
cd ..
cd ~
cd -
```

- **在干什么**：  
  - `cd 路径`：把**当前工作目录**改成指定路径；路径可以是绝对或相对。  
  - `cd ..`：切换到**父目录**。  
  - `cd ~`：切换到**当前用户家目录**。  
  - `cd -`：切换到**上一次所在目录**（在两个目录间来回切时有用）。  
- **背后结构**：只改变 shell 的「当前目录」状态，不输出内容；若路径不存在会报错。

---

### 3.2 新建目录与文件

**mkdir（Make Directory）**

```bash
mkdir mydir
mkdir -p a/b/c
```

- **在干什么**：  
  - `mkdir mydir`：在当前目录下创建名为 `mydir` 的目录。  
  - `mkdir -p a/b/c`：**递归创建**；若 `a`、`a/b` 不存在会一并创建，避免「父目录不存在」报错。  
- **背后结构**：在文件系统里新建一个「目录」类型的 inode，并在父目录中加一条对应项。

**touch**

```bash
touch newfile.txt
touch existing.txt
```

- **在干什么**：  
  - 若文件**不存在**：创建一个**空文件**。  
  - 若文件**已存在**：只更新其**修改时间**，不改变内容；常用来「刷新时间戳」或确保文件存在。  
- **背后结构**：新建时在目录里加一个 0 字节的普通文件；不负责「编辑内容」，编辑用编辑器或重定向。

---

### 3.3 复制、移动、删除

**cp（Copy）**

```bash
cp file.txt backup.txt
cp file.txt /tmp/
cp -r mydir mydir_backup
```

- **在干什么**：  
  - `cp file.txt backup.txt`：复制**文件**，得到一份内容相同的新文件。  
  - `cp file.txt /tmp/`：复制到目录，文件名不变（结果 /tmp/file.txt）。  
  - `cp -r mydir mydir_backup`：**递归**复制目录，整个目录树都拷过去。  
- **背后结构**：读源文件内容并写入新路径；不删除源。复制目录必须 `-r`，否则会报错。

**mv（Move）**

```bash
mv oldname.txt newname.txt
mv file.txt /tmp/
mv mydir /opt/
```

- **在干什么**：  
  - 同分区内：相当于改「目录项」里的名字或父目录，**不拷贝数据**，速度快。  
  - 若目标是另一分区或另一磁盘：则先复制再删除源。  
  - 也用来**重命名**：同一目录下 `mv a b` 即把 a 改名为 b。  
- **背后结构**：本质是修改目录项；所以「移动」和「改名」在同一个文件系统里是同一类操作。

**rm（Remove）**

```bash
rm file.txt
rm -r mydir
rm -rf mydir
```

- **在干什么**：  
  - `rm file.txt`：删除**文件**（不可恢复，无回收站）。  
  - `rm -r mydir`：**递归**删除目录及其下所有内容；会提示确认（若目录非空）。  
  - `rm -rf mydir`：递归删除且**不询问**（-f = force）；非常危险，尤其 `rm -rf /` 会毁掉系统。  
- **背后结构**：删除目录项并释放 inode；数据块可能仍存在直到被覆盖，但普通用户无法按文件名再访问。**使用前务必确认路径。**

---

## 4. 总结 / 与其它知识的联系

- **pwd / ls / cd** 解决「在哪、有什么、去哪」；**mkdir / touch** 解决「建目录、建空文件」；**cp / mv / rm** 解决「复制、移动/改名、删除」。  
- 路径用绝对（从 `/`）或相对（`.`、`..`、`~`）；`ls -a` 才能看到隐藏项。
