# Linux 概述与目录体系（FHS）

> 搞清 Linux 是什么、和发行版的关系，以及「根目录下那一堆文件夹各自放什么」——FHS 标准下的目录结构，为后续指令打基础。

---

## 1. 核心概念 / 定义

| 术语 | 一句话定义 |
|------|------------|
| **Linux** | 狭义指**内核**（kernel）：负责硬件驱动、进程调度、内存与文件系统管理；广义常指「基于 Linux 内核的操作系统」。 |
| **发行版（Distribution）** | 内核 + 软件包管理 + 默认软件 + 配置方式的组合；如 Ubuntu、CentOS、Debian、Arch。同一套 shell 指令在多数发行版上通用。 |
| **FHS（Filesystem Hierarchy Standard）** | 文件系统层次标准：约定根目录下各目录的**用途**，不同发行版大致遵循，便于迁移和运维。 |
| **根目录 /** | 整个文件系统的起点；绝对路径从 `/` 开始，如 `/home/user`。 |
| **一切皆文件** | Linux 把设备、进程信息、网络等也抽象成「文件」或「目录」，用读写文件的方式与系统交互。 |

---

## 2. 要点 / 步骤

1. **Linux** = 内核；日常说的「用 Linux」多半指用某个**发行版**，在 **shell** 里敲指令。
2. 根目录 **/** 下每个目录有约定用途；记牢 `/home`、`/etc`、`/usr`、`/var`、`/tmp`、`/root` 对查路径、排错很有用。
3. **绝对路径**从 `/` 开始；**相对路径**从当前目录 `.` 或父目录 `..` 开始；`~` 表示当前用户家目录。

---

## 3. 细节与推导

### 3.1 Linux 内核与发行版

- **内核**：不直接面向用户；负责 CPU 调度、内存管理、驱动、文件系统等。你装的「操作系统」= 内核 + 用户态程序。
- **发行版**：在同一个内核之上，选不同的**包管理器**（apt / yum / dnf / pacman）、默认软件和目录细节。学指令时，多数命令在各发行版上一致；装软件用哪个命令要看发行版（apt 还是 yum 等）。

### 3.2 根目录下主要目录（FHS 简表）

| 路径 | 用途 | 背后含义 |
|------|------|----------|
| **/home** | 普通用户的**家目录**，每人一个子目录（如 /home/zhang）；桌面、文档、下载等默认在这里。 | 用户私有数据与配置的归属地。 |
| **/root** | 超级用户 root 的家目录；不是「根目录」，只是 root 用户的 home。 | 避免与根 `/` 混淆。 |
| **/etc** | **配置文件**；系统与多数服务的配置（如网络、SSH、nginx）放在这里，一般不要放业务数据。 | 配置与数据分离，便于备份与迁移。 |
| **/usr** | 用户空间程序与只读数据；可执行文件多在 `/usr/bin`，库在 `/usr/lib`，头文件在 `/usr/include`。 | 早期「UNIX System Resources」的缩写；系统安装的软件主体在这里。 |
| **/var** | **可变数据**；日志（/var/log）、缓存（/var/cache）、网站数据（/var/www）、包数据库等。 | 运行时会产生变化的数据，与「只读」的 /usr 区分。 |
| **/tmp** | **临时文件**；重启后可能被清空，不要放重要数据。 | 供程序与用户放临时产物。 |
| **/opt** | 可选/第三方软件，有时一个软件一个子目录。 | 与包管理器安装的 /usr 区分。 |
| **/dev** | **设备文件**；磁盘、终端、null 等都以「文件」形式出现。 | 一切皆文件：通过读写这些「文件」与硬件交互。 |
| **/proc** 与 **/sys** | **虚拟文件系统**；反映进程与内核状态，读这些「文件」就是在查系统信息。 | 不占磁盘，动态生成；如 /proc/cpuinfo、/proc/meminfo。 |
| **/bin**、**/sbin** | 早期放**基础命令**（bin=可执行，sbin=系统管理用）；现在很多发行版用符号链接指向 /usr/bin、/usr/sbin。 | 保证在最小环境中也能用基本命令。 |

### 3.3 路径与「一切皆文件」

- **绝对路径**：从根 `/` 写起，如 `/home/user/docs/file.txt`；与当前所在目录无关。
- **相对路径**：从当前目录开始，如 `./file.txt`（当前目录下）、`../other/file.txt`（上一级目录的 other 下）。
- **~**：表示**当前用户的家目录**，如 `~/docs` = `/home/当前用户名/docs`；root 下 `~` = `/root`。
- **一切皆文件**：设备、管道、socket 在 `/dev`、`/proc`、`/sys` 里都以文件形式存在；很多「查状态」的操作就是 `cat` 或读这些路径。

## 4. 总结 / 与其它知识的联系

- Linux 体系 = **内核 + 发行版**；学指令主要和 **shell + FHS 目录** 打交道。
- 根目录下 **/home、/etc、/usr、/var、/tmp、/root** 先记牢，再扩展 /dev、/proc、/sys；后续 `cd`、`ls`、`find`、配置路径都会用到。
