# 查看、编辑与文本处理

> 不打开图形编辑器的情况下：怎么看文件内容（cat / less / head / tail）、怎么搜内容（grep）、以及用 vim 做最基础的编辑；每条命令在干什么、适用场景是什么。

- **标签**：`#Linux` `#cat` `#less` `#grep` `#vim` `#文本`
- **日期**：2025-03-11
- **来源**：Ad-Studium 学习笔记

---

## 1. 核心概念 / 定义

| 术语 | 含义 |
|------|------|
| **标准输出（stdout）** | 命令把结果打印到终端；可被重定向或管道接到下一个命令。 |
| **分页器（pager）** | 一屏一屏显示内容，如 less、more；大文件用它们避免刷屏。 |
| **正则 / 模式** | grep 用「模式」在每行里匹配；支持简单正则，便于搜关键词。 |

---

## 2. 要点 / 步骤

1. **整文件一次看完**：`cat`；**大文件逐屏看**：`less`（可前后翻，推荐）。  
2. **只看头/尾几行**：`head`、`tail`；**tail -f** 可持续看日志追加。  
3. **在文件里搜包含某串的行**：`grep`。  
4. **简单编辑**：`vim` 或 `nano`；vim 至少会 `i` 插入、`Esc`、`:wq` 保存退出。

---

## 3. 细节与推导

### 3.1 查看内容

**cat（Concatenate）**

```bash
cat file.txt
cat file1.txt file2.txt
```

- **在干什么**：把**一个或多个文件**的内容依次输出到终端（标准输出）。  
- **背后结构**：不分页，全部一次性打出；文件很大时终端会刷很多屏。适合小文件或配合管道给下一个命令用（如 `cat file | grep "key"`）。  
- **注意**：不要用 `cat` 看二进制文件，会乱码或把终端搞乱。

**less**

```bash
less file.txt
less +F /var/log/syslog
```

- **在干什么**：**分页**显示文件；可前后翻（空格下一页、b 上一页、方向键、PgUp/PgDn），支持搜索（`/关键词` 然后 n 下一个）。  
- **背后结构**：不会一次性把整个文件读进内存，大文件也适合。  
- **常用**：`q` 退出；`/word` 搜索 word，`n` 下一个匹配；`less +F` 类似 `tail -f`，盯住文件尾部追加。

**head 与 tail**

```bash
head -n 20 file.txt
tail -n 20 file.txt
tail -f /var/log/app.log
```

- **在干什么**：  
  - `head -n 20`：只输出文件**前 20 行**（-n 可写为 -20）。  
  - `tail -n 20`：只输出**最后 20 行**。  
  - `tail -f`：持续监视文件，**追加**的新内容会实时显示；看日志常用，Ctrl+C 退出。  
- **背后结构**：head 从头读；tail 从尾读（或从末尾往前数行），-f 时保持文件打开，读到新写入就输出。

### 3.2 在内容里搜索：grep

```bash
grep "error" file.log
grep -i "error" file.log
grep -r "function" /path/to/code
grep -n "keyword" file.txt
```

- **在干什么**：在**每一行**里找是否包含给定**模式**（默认是普通字符串），匹配的行整行输出。  
  - `-i`：**忽略大小写**。  
  - `-r`：**递归**在目录下所有文件里搜。  
  - `-n`：输出时带**行号**。  
- **背后结构**：按行读文件，用模式匹配；支持基本正则（如 `^` 行首、`$` 行尾、`.` 任意字符）；复杂正则可用 `-E`（扩展正则）或 `egrep`。  
- **常见用法**：`ps aux | grep nginx`、`cat file | grep "key"`，把上一命令输出当「文件」来搜。

---

### 3.3 最简 vim 用法（能编辑、能保存）

vim 是模态编辑器：**正常模式**（默认）下按键是命令，**插入模式**下才是打字。

| 操作 | 按键 / 命令 |
|------|-------------|
| 打开文件 | `vim file.txt` |
| 进入插入模式（开始打字） | 按 **i**（或 a 在光标后插入） |
| 退出插入模式 | 按 **Esc** |
| 保存 | **:w** 回车 |
| 退出 | **:q** 回车 |
| 保存并退出 | **:wq** 回车 |
| 不保存强制退出 | **:q!** 回车 |

- **在干什么**：`:w` 把缓冲区写回文件，`:q` 退出；未保存就 `:q` 会提示，加 `!` 强制。  
- **背后结构**：vim 把文件读进内存（缓冲区），编辑在内存里进行；只有 :w 才写回磁盘。  
- 若只想快速改一两行，也可用 `nano file.txt`，底部有提示，Ctrl+O 保存、Ctrl+X 退出。

## 4. 总结 / 与其它知识的联系

- **cat** 全量输出；**less** 分页看大文件；**head/tail** 看头尾；**tail -f** 盯日志。  
- **grep** 在行内搜模式，常和管道一起用（见 06）。  
- **vim** 至少掌握：i → 编辑 → Esc → :wq；复杂编辑可后续再学。下一篇讲**进程与系统**（ps、top、kill、systemctl）。
